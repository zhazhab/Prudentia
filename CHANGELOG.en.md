# Changelog

[中文](CHANGELOG.md)

All notable changes to Prudentia should be recorded here. Add the newest entry at the top of the current release section.

## Unreleased

- Refactored the full conversation runtime: the `ConversationEngine` entry point now keeps only run, thread, and confirmation-action APIs, while subject resolution, research, generation, and action extraction live in a dedicated runtime module. Each turn freezes a `TurnContext` (effective subject, request, research plan, and model tier) and a `StepContext` (the exact model context and persisted sources), preventing later stages from resampling mutable state. Strongly typed `ConversationEvent` values generate the existing WebSocket event names and JSON without changing the wire contract. A cooperatively cancellable `TurnTask` replaces raw `JoinHandle` storage, propagating one cancellation signal through research, visible generation, and action extraction before using forced abort as a timeout fallback. Terminal run updates are conditional, so only the first of completed, failed, canceled, or interrupted can win and a canceled run cannot later appear completed. Subject-candidate sorting and company-suffix stripping now use equivalent forms accepted by Rust 1.96 Clippy, preserving strict CI compatibility.
- Expanded the execution and evidence budgets for long turns so deep company analysis is not cut off by short safety thresholds. Visible-response hard limits are now 90, 240, and 600 seconds for simple, standard, and deep work, while confirmation-card extraction allows 120 seconds after simple/standard turns and 300 seconds after deep turns; normal completion and user cancellation still end immediately. The response model can now receive up to three sources per evidence tier, with up to 8,000 characters for each primary source and 2,500 for each independent/community source. Storage remains bounded at nine sources per turn, 8,000 characters per source, and a 24-hour research cache.
- Upgraded focused moat analysis from a summary checklist to a decision-auditable judgment. The core now targets roughly 3,500-4,500 Simplified Chinese characters, and every mechanism must retain dated evidence, causal chain, competitor comparison, maintenance cost, counterevidence, confidence, and upgrade/downgrade/kill conditions. A 1-5 score or unsupported half-point is forbidden without observable anchors. Risk, competition, counterevidence, and "how to breach it" wording remain internal dimensions of the focused moat audit instead of incorrectly selecting the broad-company template. Only the three to five breach paths ranked highest by probability times impact remain; every path uses the same explicit fields for the attacker, cheapest sequence, required resources, company response, economic-profit transmission, leading indicators, disconfirming condition, missing evidence, and verdict-change point. If space is tight, the model must include fewer paths rather than compress or omit the final dossier. Company-view projection no longer has a contradictory 500-character limit: moat, business-quality, financial, and other sections are capped at 3,500, 3,000, 2,000, and 1,200 characters, with a 9,000-character total per patch; repeated prose is compressed before decision evidence. Deep turns now allow 90 seconds for confirmation-card extraction while simple and standard turns remain at 45 seconds. Moat research now also seeks retention, repeat purchase, merchant economics, contribution margin, marketing efficiency, quality/refund, and competitor-response evidence.
- Fixed natural lowercase ticker references in general threads failing to bind a company and therefore skipping research on detailed follow-ups. Explicit investment/analysis/research wording such as `I want to invest in pdd` can resolve and bind one unique pure-letter ticker, while isolated greetings such as `hi` still remain ordinary conversation.
- Fixed the recent-thread action menu being clipped when only a few threads existed. The thread list now fills the remaining rail height and aligns entries at the top, keeping Archive and Delete visible while preserving scrolling for long lists.
- Company questions that explicitly say to use only existing/local context now skip external retrieval and use the standard model, so runtime status no longer claims to build a multi-year financial series. Reloading Home now always opens the latest active thread instead of restoring the last manual selection.
- Fixed real Codex CLI turns that completed visibly but failed to produce a confirmation card. Conversation action projection now uses the CLI's native JSON Schema to enforce the `summary` / `actions` envelope, then validates and decodes action payloads before domain code sees them. Malformed output remains a run warning and can never mutate a company view, trade, or investment rule directly.
- Fixed ordinary punctuated conversation and instructions such as “update the company view” being misread as a company abbreviation. Company research starts only from an explicit company-analysis structure or one high-confidence candidate. A pending clarification now accepts only a candidate ordinal, security code, or short company name; a new complete request supersedes it.
- Company threads with a confirmed company view now open the right context rail on Company View by default and reveal a newly confirmed version immediately. Sections use safe Markdown rendering and open questions render as a list.
- Fixed research-cache work and persisted run-progress updates waiting on one another with a single-connection database pool. Research now advances independently and is aborted with its parent run on cancellation or interruption.
- Made company-short-name resolution fail closed. Holdings names, the bound thread company, security codes, and the local symbol directory now feed one candidate resolver, and research starts only for a single high-confidence candidate. Multiple candidates or an unrecognized explicit company request no longer fall back to the thread's company; a light real-AI turn asks the user to confirm the full company name or security code. That clarification turn performs no external research, receives none of the bound company's summary, view, sources, or rules, and cannot replace the bound thread summary. The original request and candidates are persisted with the message, so a code or candidate ordinal resumes the original request after refresh; a still-ambiguous reply asks again. The frontend explains the `subject_clarification` route reason.
- Fixed cross-company contamination inside company threads. For example, asking “analyze Tencent” in a PDD-bound thread now resolves `0700.HK / Tencent Holdings` from held-company aliases and uses Tencent for the turn's research plan, sources, company context, and confirmation actions while leaving the persisted thread subject bound to PDD. The turn no longer receives PDD's thread summary, prior-turn summaries, or company view. Turn summaries are stored and loaded by effective company, so a Tencent summary cannot replace or contaminate the PDD thread summary, and research query/cache identities remain company-specific.
- Fixed long conversations becoming unusable at medium and mobile widths: the `901–1250px` layout no longer stacks the right-side Company View below the chat, retaining the thread rail and opening context in a right-side drawer instead; the mobile chat workspace now stays within the available viewport, scrolls history inside the message list, and keeps the composer visible. Company-view, trade, and rule confirmation cards now follow the assistant reply that proposed them instead of accumulating at the end of the entire conversation; applied/rejected cards collapse to a compact state, and action-type labels follow the UI language. Opening or switching threads now starts at the latest message, while later renders preserve deliberate upward history scrolling.
- Fixed detailed company analysis still squeezing the business model despite placing it first. Broad company analysis now targets roughly 4,000-5,000 high-density Simplified Chinese characters with at least 60% of substantive content reserved for the business model; focused business-model analysis targets 3,500-4,500 characters with at least 75% reserved for the core. The framework now traces offering/customer job, transaction lifecycle and risk ownership, monetization/profit pool, segment-level unit economics, competition and attacker economics, positive compounding conditions, and inverse failure plus multidisciplinary synthesis; materially different products/geographies stay separate and every section distinguishes fact, inference, and unknown. Detailed mode is no longer overridden by the global concise-chat instruction. Light/standard limits remain 45/90 seconds while deep visible responses move from 150 to 240 seconds. A real PDD UAT produced 8,523 characters, including 5,678 business-model characters (66.6%), in 126.7 seconds.
- Closed the Buffett/Munger company-quality gaps in operating analysis. Every business-model, moat, and broad company analysis now applies a knowability gate and classifies the enterprise as predictable, partially predictable, or not predictably bounded; a historical financial base no longer forces numerical five/ten-year extrapolation. Generic detailed/full/direct company-analysis wording also enters the complete framework. Added explicit owner-economics/reinvestment, management/culture/incentives/capital-allocation, and financial-resilience/ruin-risk audits covering per-share economics, maintenance versus growth capex, dilution, incremental ROIC, return on retained earnings, reinvestment runway, and great/good/gruesome classification. SEC Company Facts now adds capex, share compensation, diluted shares, a free-cash-flow proxy and per-share proxy with structured units, never labels the proxy owner earnings, and removes/flags an isolated 50x annual scale outlier. Filing excerpts add matching evidence windows; Tavily queries use category focuses; every source snippet is capped at 8,000 characters before cache/storage; and the public-sources cache version advances. Deep replies remove repeated methodology while keeping hard decisions. Confirmation extraction uses the standard model, and only investment-system subjects may propose deep rule graphs. Company response/projection context deterministically removes historical valuation, trades, and the investment system; company patch preparation and execution also remove valuation changes. CLI and OpenAI-compatible structured replies use `serde_json` to find the first complete target object despite fences or trailing commentary.
- Scoped every company conversation to enterprise operations. Model context no longer includes portfolio summaries, positions, trade history, or the investment system. Visible replies and company-view confirmation cards exclude share prices, quotes, market capitalization, valuation multiples, price targets, stock returns, technical charts, ratings, personal portfolio P/L, and trading conclusions. Automatic company patches leave `valuation_expectations` unset, while `thesis` means the operating company thesis only. Product markets, industry structure, competitors, and market share remain in scope. Research plans no longer request capital-market evidence; Yahoo and TradingView results that only discuss stock valuation, prices, or trading technique are filtered, and the public-sources cache version now reflects the company-operating scope.
- Added a shared mandatory investor questionnaire to business-model, moat, and broad company analysis. Every analysis must answer what the company provides and why it is hard to substitute; whether profit is economically real and sustainable; the recurring cost of maintaining the advantage; the number and threat of serious competitors/substitutes; the capital, time, capabilities, distribution, and three-to-ten-year success range for a rational attacker; and five-to-ten-year company-operating downside/base/upside earnings power. Operating cases use company variables such as demand, share, competitive spending, revenue, margins, reinvestment, regulation, and capital intensity, and explicitly exclude stock-market bear/bull conditions, share-price paths, and valuation multiples. Replies end with a fixed six-row decision matrix, and each row presents verified facts, optimistic case, pessimistic case, current verdict, and missing evidence. When a normalized base is available, the model must calculate both five- and ten-year operating-profit or owner-earnings ranges rather than stopping at growth and margin assumptions. Accounting profit is reconciled with cash flow, maintenance capex, stock compensation, working capital, subsidies, and one-offs. Replacement costs, odds, and forward earnings use ranges only when supported by facts, comparable cases, or explicit assumptions; otherwise the answer stays qualitative. Company response structures now live in a dedicated module, while projections separately retain non-substitutability/profit authenticity, moat maintenance and attacker economics, and long-range earnings scenarios. Visible-response timeouts are 45, 90, and 240 seconds for light, standard, and deep tasks respectively; action projection remains capped at 45 seconds.
- Deepened business-model analysis with inversion and a multidisciplinary latticework. Focused business-model questions now use the deep model and a dedicated structure covering mechanics/economics, positive conditions, failure architecture worked backward from permanent loss, three to five causally relevant disciplinary lenses, and an integrated robustness verdict. Available lenses include microeconomics/industrial organization, accounting and corporate finance, psychology/incentives, systems thinking, game theory, technology/operations, organization/regulation, and outside-view base rates; each must provide model -> observed mechanism -> required evidence -> implication, preventing vocabulary-only analysis. Research plans now seek cash conversion, return on invested capital, customer/supplier incentives, competitor response, and bottleneck evidence. Company views retain positive conditions, inversion paths, earliest break signals, and kill conditions.
- Converted the user-provided moat reasoning into an executable moat audit: product quality, market share, management/execution, founder talent, marketing/distribution, and temporary technology leadership no longer qualify automatically. Candidate mechanisms must prove the chain mechanism -> customer or cost behavior -> competitor constraint -> retained excess returns and survive counterfactual tests for subsidy removal, replicability, low switching costs, founder departure, channel shifts, and technology change. Focused moat questions now use the deep model and a dedicated response structure; research queries seek pricing power, network effects, scale economies, switching costs, protected IP/scarce resources, and durability evidence. Confirmation cards retain strength, horizon, evidence, and kill conditions without adopting the source article's company-specific opinions.
- Deepened company business-model analysis: it must separately rate competitive intensity (low/medium/high), relative competitive position (weak/mixed/strong), and the difficulty of earning durable profits (easy/moderate/hard), grounded in competitors/substitutes, switching and multi-homing, stakeholder bargaining power, profit pools, pricing power, acquisition and marketing, subsidies, fulfillment, reinvestment, and incremental margins. SEC annual-report research now extracts separate business, competition, monetization, and profit-engine windows; the five-year XBRL series adds gross profit, cost of revenue, operating income, and selling-and-marketing expense. The first company-view section is now “Business Model, Competition, and Profit Quality.”
- Made broad company analysis business-model-first: responses now begin with the product/service, users/customers/payers, value chain, goods/data/cash flows, revenue streams, cost structure, working capital, capital intensity, and unit economics before assessing business quality, moat, financial validation, valuation, risks, and investment view; the investment conclusion no longer precedes the operating analysis. Broad and business-model research also includes a five-year SEC XBRL financial series by default.
