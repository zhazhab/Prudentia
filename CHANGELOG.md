# 变更日志

[English](CHANGELOG.en.md)

Prudentia 的重要变更都应记录在这里。最新条目插入当前版本段的最上方。

## 未发布

- 修正对话输入框的中文输入法交互：候选词组合期间按 `Enter` 只确认上屏，不再误发送消息，并兼容 `keyCode=229` 的浏览器事件。
- 优化对话完成时机与上下文展示：无附件、无检索的纯寒暄/能力询问仍由真实 AI provider 回复，但不再启动冗余的结构化动作提取；前端将线程摘要、历史轮次、持仓和规则图等内部上下文标签显示为可读的本地化文案。
- 将首页临时 memo chat 重构为完整 `ConversationEngine` 工作区：真实 CLI/OpenAI SSE provider、有序非 mock fallback、持久化运行事件与断线续传、真实阶段/取消/重试、线程主题与移动抽屉、Markdown/来源/附件/研究降级，以及逐项确认的公司看法版本、历史汇率交易账本和可执行规则 DAG；共享附件和公司 Markdown 使用相对路径写入原仓库本地根目录。
- 对话等待期间不再只显示省略号：前端区分“正在连接后端”“后端正在运行”和“后端正在流式返回”，并持续显示已运行时长。
- 修正持仓市值排序口径：`GET /api/portfolio/positions` 返回单只持仓 CNY 市值 `market_value_base`，后端默认顺序和前端“市值”排序都按统一 CNY 价值排序，不再按本币直面数字比较。
- 新增 Portfolio 自动交易调整和组合时间加权收益率：导入、草稿确认、持仓编辑或删除导致 CNY 组合市值变化时自动记录买入/卖出调整并用于组合收益率，收益响应保留未调整交易变动的快照收益率用于解释；单只持仓收益仍按券商浮盈亏口径。
- 持仓表支持按市值、盈亏、收益率、周期收益率和权重排序，默认按市值降序展示，并在本地保存排序规则。
- 将 Portfolio 收益率类展示统一保留两位小数；持仓权重仍保持原有显示口径。
- 持仓表新增当前收益率和跟随 Portfolio Performance `本月` / `本年` / `记录起` 的周期收益率；后端新增持仓快照表，`GET /api/portfolio/positions?period=...` 返回单只持仓的周期 CNY 收益和收益率。
- 统一 Portfolio 金额展示为 ISO 币种前缀格式，例如 `CNY 1,234.56`、`HKD 1,234.56` 和 `USD 1,234.56`。
- 将上证 benchmark 从 ETF 代理 `510210.SS` 改为官方上证综指 `000001.SS`，并让腾讯行情 provider 支持指数代码 `sh000001`；读取 benchmark 时按当前 symbol 过滤旧快照，避免新旧口径混算。
- 为持仓刷新补充券商口径回归测试：本币市值按 `last_price × quantity`，本币浮盈亏按 `(last_price - average_cost) × quantity`，CNY 汇总再按 FX 转换。
- 将 Portfolio Performance 的 benchmark 快照收敛到持仓价格刷新周期：导入、编辑、删除只写组合快照，benchmark 只在同一轮 `price_refresh` 中和持仓行情一起刷新。
- 修正 Portfolio Performance 基准快照：`mock` 行情不再被当作可用指数代理数据写入或读入收益曲线，而是标记为 stale/unavailable，真实收益跟踪文档示例改为 `yahoo,tencent` 或 `longbridge,yahoo`。
- 将不同 git worktree 的本地 `.env` 和默认 SQLite 数据库统一指向 Git common dir 所在的原仓库工作目录，避免新 worktree 启动时读到空配置或空持仓；`PRUDENTIA_LOCAL_DIR` 可覆盖该位置，相对 SQLite URL 会相对该目录解析。
- 新增行情刷新频控：真实 market data provider 会经过 provider 级最小请求间隔、429/频控冷却和 fallback 降级；持仓与 benchmark 报价改为 batch 优先，腾讯行情和长桥支持同轮批量获取；持仓页新增手动强制刷新入口，后台自动刷新仍默认每日 TTL。
- 扩展 MarketDataProvider：`MARKET_DATA_PROVIDER` 支持逗号分隔 fallback 链，并新增 Yahoo Finance、腾讯行情和长桥 OpenAPI provider；长桥通过 `LONGBRIDGE_APP_KEY`、`LONGBRIDGE_APP_SECRET` 和 `LONGBRIDGE_ACCESS_TOKEN` 读取凭证。
- 在 Portfolio Performance 的指数代理状态中显示行情来源，并在 benchmark API 中返回 `source`，避免 mock 数据被误认为真实市场数据未更新。
- 修正 Portfolio Performance 在清空持仓后重新导入的收益口径：删除到空仓会作为新的快照起点，避免把清理脏数据后的重新导入误算成正收益。
- 调整首页 memo chat 为自然对话优先：普通输入走 provider-backed 对话回复，只有用户明确要求保存、记录或整理成备忘录时才进入 memo extraction 并生成 `memo_draft` artifact；`你好`、`hello` 等轻量寒暄直接返回简短对话。
- 将前端默认入口改为 chat-first 投资备忘录首页：左侧展示最近 memo threads，中间通过 AI WebSocket 流式对话并支持同一发送按钮终止任务，右侧以独立持仓星图展示当前组合上下文；新增 `memo_threads` / `memo_thread_messages` 持久化、memo thread REST API 和 `memo_chat.start` WebSocket 协议。
- 移除 Portfolio 主界面的市场配置面板：持仓页只保留收益表现、核心统计、指数对比、导入和持仓表。
- 精简 Portfolio 持仓展示：移除顶部价格状态卡、持仓表“状态/待刷新”列和市场配置里的“汇率待刷新”标签，不再在主界面展示行情新鲜度状态。
- 调整 Portfolio Performance 默认视角：周期收益默认展示百分比，仍可切换到金额。
- 精简 Portfolio 顶部统计：移除“市场分组”统计卡，并去掉价格状态卡中的“汇率待刷新”说明，避免重复展示不必要的行情状态。
- 增强 Portfolio Performance 指数对比：新增年化收益字段和顶部年化收益卡片，指数代理图支持累计收益、年化收益、相对指数超额收益三个对比维度。
- 优化 Portfolio Performance 折线图展示：当当前周期只有一条快照时显示明确的起点标记和说明，并动态计算 Y 轴范围，避免单点 0% 数据看起来像空图或异常坐标轴。
- 新增 Portfolio Performance 快照视图：支持 `本月`、`本年`、`记录起` 和 `金额/百分比` 切换，使用组合 CNY 市值快照计算周期收益，并用 `SPY`、`2800.HK`、`000001.SS` 作为标普/恒生/上证 benchmark 对比；导入确认、编辑、删除和每日行情刷新会写入快照，前端通过读取本地 API 自动更新。
- 修正 Portfolio 草稿缺少代码时的确认失败：匹配代码和确认导入都会先按名称、市场和币种继承当前已有持仓的唯一 `symbol`，再查本地代码库，避免已存在持仓的截图简称仍阻断导入。
- 修正 Portfolio 截图导入市值计算：识别到现价/当前价时优先按 `last_price × quantity` 计算本币市值，不再把可能是基础币种口径的截图市值列反推为股票现价；默认 mock 行情刷新也不会覆盖导入市值，并会用 notes 中的 `current_price` 修复已被旧 mock 覆盖的持仓。
- 修正 Portfolio 草稿自动整理：清除旧版重复代码错误，并将 `HK`/`港股` 等市场别名按同一市场归并，避免让用户手动整理可自动判断的重复持仓。
- 修正 Portfolio 多截图导入：草稿中相同 `symbol` 的行会自动归并，数量相加、平均成本按数量加权、市值相加，避免两个 `0700.HK` 阻断确认。
- 移除 Portfolio 导入工具栏中的“粘贴截图”入口，截图导入统一通过“新增文件”选择图片。
- 优化 Portfolio 导入工具栏：移除顶部“新增行”大按钮，将手填入口收进草稿表右上角的小 `+` 图标，导入动作区更简洁。
- 优化 Portfolio 本地代码匹配：短港股数字代码会规范为内部 `0700.HK` 形式，搜索 `700` 或草稿确认 `700` 可以匹配/保存为 `0700.HK`。
- 修正 Portfolio 导入草稿代码输入框：草稿行现在使用稳定的前端行标识，编辑 `symbol` 时不会重建整行，连续输入不会被打断。
- 收缩 SQLite `security_symbols` 表结构：只保存 `symbol/name/market/currency/updated_at`，启动迁移会移除旧表中的 provider、exchange 和 asset type 列；symbol search API 也只序列化 lookup 字段。
- 继续精简 Portfolio public 默认代码库存量文件：`symbols.json` 中的 source 只保留 `id/count`，证券记录只保留 `symbol/name/market/currency`，不再写入 provider、exchange 或 asset type。
- 清洗 Portfolio public 默认代码库存量数据：生成 `symbols.json` 前会将公开源中的繁体中文证券名称统一转换为简体。
- 精简 Portfolio public 代码库默认存量文件：`symbols.json` 只保留文件级 `updated_at`，单条证券记录不再写入更新时间；导入 SQLite 时统一使用文件级时间戳。
- 将 Portfolio public 代码库默认数据源改为项目内标准化存量文件 `data/symbol-directory/public/symbols.json`：启动先导入该文件，按 `updated_at` 判断 24 小时过期后才异步刷新公开源并覆盖存量文件，刷新失败只记录错误且不阻塞启动。
- 将 Portfolio public 证券代码源配置化：公开源 URL、解析类型、缓存文件、缓存目录和 24 小时过期时间统一放在 `config/symbol-directory-public.json`，后端刷新时按需读取，缓存过期后再更新源文件。
- 新增 Portfolio 本地证券代码库：默认 public provider 会免账号后台刷新上交所、HKEX 英文/繁体中文和 Nasdaq Trader 公开目录，源文件缓存在 `data/symbol-directory/public/` 并以 24 小时过期；中文名称匹配会做简繁折叠；截图识别和草稿确认只查本地 `security_symbols` 补 `symbol`，导入面板保留“匹配代码”用于已有草稿重跑本地匹配，Tushare/授权源作为可选扩展。
- 调整 Portfolio 导入草稿：打开导入工具和清除草稿不再自动插入空白行；手填必须通过“新增行”，截图结果会直接追加到空草稿。
- 收敛前端功能面：当前前端只保留 Portfolio 持仓/导入、Memo 和 AI 设置，移除总览、Decision Delta、Research、投资体系和画像页面及其前端 API/type/i18n 表面。
- 清理前端旧接口残留：移除不再使用的一步 Portfolio 导入 API wrapper、旧导入/截图预览文案、旧 AI settings 文案和孤立请求类型，保持当前 UI/API 表面只包含已接通能力。
- 优化 Portfolio 导入入口：CSV/TSV/XLSX 和截图统一通过“新增文件”选择，前端按文件类型分流到 CSV/Excel 预览或截图识别。
- 精简 Portfolio 导入草稿表：只展示 `代码`、`名称`、`数量`、`平均成本`、`币种`、`市场` 六个核心字段，隐藏来源、账户、行业、市值、备注、置信度和问题列。
- 修正 Portfolio 草稿确认：非空草稿行缺少 `symbol` 时会在前端以中文行内错误阻断确认，避免把 provider 解析失败的英文错误暴露给用户。
- 调整 Portfolio 导入工具：手填入口改为显式“新增行”，不再自动创建占位草稿行。
- 修正 Portfolio 截图导入：提示词改为通用规则，保留持仓表中有数量、成本/现价、市值或盈亏指标的现金命名 ETF/基金/证券行；无法识别代码时要求用户在草稿表补齐。
- 补充 `AGENTS.md` 开发约束：新开发任务默认新建 worktree，开发中跑最小相关检查，`fmt`、文件长度和 clippy 保留到最终合入/发布前或由 CI 覆盖；每次改完判断并同步项目文档/changelog，完成后启动前后端等待验收，且仅在用户明确要求后提交、推送或发起 PR。
- 新增 `AGENTS.md`，记录默认收尾流程：每次实现或调整后运行验证、重启 `make start`、检查后端健康状态并回报前后端地址。
- CI 新增 `cargo clippy --workspace --all-targets -- -D warnings`，并提供本地 `make check-backend-clippy` 命令。
- 修正 Portfolio 截图导入：名称包含“现金”的 ETF/基金类持仓如果有数量、成本和市值等持仓指标，会保留为草稿行，不再被当作纯现金余额过滤。
- 新增后端 Rust 单文件 800 行限制，并将 Portfolio 与 Decision Delta 后端实现按类型、路由、导入、持久化、刷新等逻辑拆分。
- 改进开发启动脚本：当前后端默认端口被占用时，`./scripts/dev.sh` 会自动选择可用端口，并把前端连接到实际后端地址。
- 优化 Settings 页 AI 配置：按 Mock、OpenAI-compatible、CLI provider 仅展示当前方式需要的输入项，并在保存时持久化配置。
- 新增统一 AI WebSocket 通道，并将 Portfolio 截图识别改为可取消的多图任务队列；导入草稿现在支持默认手填、来源列、空白行过滤和重复代码归并。
- 保留 Decision Delta 后端/API 基础与性能优化：timeline 改为批量查询，detail snapshot 默认限制最近 90 条，refresh 复用 quote/FX，并新增关键 SQLite 索引；相关前端页面暂不暴露。
- 保留 Decision Delta 后端模型：为 buy/add、sell/trim、watch/skip 决策生成 actual leg 与 baseline shadow leg，并支持按最新行情刷新快照；相关前端页面暂不暴露。
- 打磨 Portfolio 持仓体验：CSV/Excel/截图统一进入可编辑草稿表，确认后按 `symbol` 合并写入；新增持仓编辑/删除、US/HK/CN 市场推断、CNY 口径汇总，以及 market data provider FX 刷新和 stale fallback。
- 保留 Research Library 后端/API 基础，支持文章/人物投资思想蒸馏、股票快照分析和持仓组合复盘；相关前端页面暂不暴露。
- 在 README 中补充仓库名解释、愿景、理想目标和计划支持功能。
- 将双语文档拆分为独立文件：简体中文保留 `.md`，英文使用 `.en.md`。
- 将变更日志改为英文和简体中文双语可用。
- README 已增加简体中文，与英文并列维护。
- 记录 changelog 排序规则：最新条目插入当前版本段最上方。
- 记录开发完成规则：每次开发后更新 changelog；当启动方式、能力、公共接口或工作流变化时同步更新 README。
- 新增工程风格规范，覆盖可读性、维护性、可解释性、Rust 设计实践、泛型、enum、trait、注释和 review 要求。
- 新增 Codex CLI 作为第一个 CLI AI backend，并提供 `codex login --device-auth` device-code 登录说明。
- 新增 provider-based AI 与 market data 边界，包括 mock provider、OpenAI-compatible AI、Alpha Vantage-compatible market data，以及可复用的 CLI AI provider 层。
- 搭建本地优先的 Prudentia 项目框架，包括 Rust 后端、SQLite 持久化、React + Vite 前端、Portfolio 导入、Memo 工作流、AI 设置和双语 UI。
