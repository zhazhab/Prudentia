# 变更日志

[English](CHANGELOG.en.md)

Prudentia 的重要变更都应记录在这里。最新条目插入当前版本段的最上方。

## 未发布

- 修复终止公司研究后刷新页面丢失计划，以及进度分母随跳过步骤缩小的问题：任意运行快照都会从持久化事件重建 `execution_plan`，终态中的运行/待执行步骤稳定映射为失败/无需执行；进度始终按完整计划步骤数展示，避免列表有七步却显示“3/4”。
- 修复证券目录后台刷新与对话重试并发时偶发 `database is locked`：创建对话运行现在会在读取线程状态前先取得 SQLite 写保留锁，避免延迟事务在已有后台写入者时发生不可等待的读写升级冲突；重试会等待短暂写入完成后原子创建新消息与运行。
- 公司研究现在把“识别公司”和“确认研究范围”分开处理：本地证券目录中唯一且处于受限编辑距离内的英文公司拼写错误会自动绑定，只有并列候选等真实歧义才询问用户。每个实质性公司运行都会持久化 `company_analysis_v1` 研究执行计划，包含研究范围、默认分析维度和逐步状态；前端展示完整计划而不是只显示 Agent 的内部轮次，刷新、断线和终止后也可按事件恢复真实进度。用户没有额外限定时直接使用综合公司分析模板，明确询问商业模式、护城河、财务、风险等主题时显示对应专项范围，但不会为了确认范围阻塞检索。
- 修复公司确认后不自动研究的问题：存在待确认公司时，“是的/对/没错/yes”等明确肯定只会在候选唯一时完成绑定并恢复原始问题；如果确定性确认最初没有候选，但最近一条 AI 确认问句唯一指向本地证券目录中的公司，也可从被终止的确认回复中安全恢复。恢复后的实质性公司问题照常自动生成官方、独立和社区三类研究计划，无需用户额外要求搜索。证券目录的 ETF 判断改为独立词或代码词尾匹配，避免因 `Netflix` 等名称内部偶然包含 `etf` 而被错误过滤。SEC、Yahoo 和 TradingView 等公共资料请求在连接、超时、响应体、429 或服务端错误时会短暂等待并重试一次，避免单次网络抖动立即降级。前端在正文生成后改为显示“正在检查本轮是否需要更新备忘录”，不再暴露“正在保存对话”的内部持久化阶段。
- 将内置公司分析 Prompt 重构为可执行的证据契约：商业模式、护城河、公司综合分析和反方分析分别使用独立类别 Schema，不再共享泛化的 `summary/findings` 模板；每项结论必须标记事实/推断/假设，并返回证据充分性、逐条来源与日期、因果链、最强反论、未知项、领先指标、证伪条件和决策影响。通用 Skill/Agent 协议改为 Schema 感知，避免与自定义产物字段冲突；Agent 只为一个会改变结论的证据缺口调用研究 Tool，社区观点只作待核验线索。证据不足时不再为了覆盖分类生成完整伪分析，只返回简短拒绝、关键缺口和最多两个待验证假设。后端会拒绝上下文中不存在的来源 URL，以及没有引用任何来源却标为事实的结论；前端展示证据等级和上述判断字段，同时兼容历史字符串证据产物。
- 修复真实 Codex CLI 执行 Agent 时因开放式 `arguments` / `output` 对象不符合严格结构化输出约束而失败的问题：Agent 决策 Schema 现在按其白名单 Tool 输入与最终产物 Schema 动态生成严格联合类型，运行时继续校验实际选择，既允许多种 Tool/最终结果形状，也不放宽额外字段。
- 优化 Capability 协作与长期运行成本：`kind`（Skill/Agent）和 `stage`（analysis/challenge）改为独立契约，公司深度分析 Agent 属于主分析阶段，只有反方 Agent 属于挑战阶段。主分析 Agent 已预加载的方法默认不再作为独立 Skill 重复执行，用户通过 `@skill_id` 点名时才额外运行，减少重复深度模型调用。Agent 依赖必须覆盖其全部 surface/subject，预加载 Skill instructions 合计限制为 48,000 字符。同一运行内基础研究与 Agent 补充研究按 URL 复用来源行和来源事件，不重复进入最终模型上下文；前端以较新的事件或 REST 快照为准，并清除已结束调用及过期的嵌套 Tool 状态，避免刷新竞态让页面持续显示运行中。
- 将 Capability 中混合的 Skill/Agent 语义拆成可持续扩展的三层运行时：Rust `native` Tool 负责确定性只读基础设施，Skill 是可直接运行或预加载的版本化分析方法，Agent 是持有精确 Skill 依赖、只读 Tool 白名单和最多八轮预算的独立模型会话。新增内置公司深度分析 Agent；公司分析与反方 Agent 可根据证据缺口自主选择公司资料或社区观点 Tool，每个 Agent 最多四次不重复调用。所有 Tool 仍由 Registry 校验精确版本、Schema、副作用、确认策略、超时和输出，模型不能扩展权限或执行写操作。Agent 当前轮次和嵌套 Tool 通过持久化事件实时展示，完成后只保存轮次、工具、状态和来源数量的有界轨迹；观察限制为 96 KiB、单次最多十二个来源且摘要截断为 1,600 字符，避免长期重复存储研究正文。能力列表 API 同时公开 Agent 已加载的 Skill 和允许的 Tool，但不公开 instructions。
- 加固 Capability 运行与刷新恢复：活动运行 API 现在返回所有未完成的并行能力，前端可合并先到的事件与后到的 REST 快照；WebSocket 会分批追完超过 2,000 条的事件积压。工具与运行失败只向 API 暴露稳定安全错误类别，原始 provider/CLI 异常仅写服务端日志。规则图新增 64 节点、256 边、8 个模型节点、512 KiB、600 秒和 500 条执行记录上限，激活前校验 Capability 参数和显式 `rule_graph_input` 权限，并固定 `manifest_hash` 防止同版本清单原地修改后改变执行语义；执行输入只存一次，节点 trace 使用引用避免重复占用。纯社区请求继续走社区专项工具，混合商业模式、财务或护城河请求保留完整公司研究证据。
- 将对话工具重构为统一的 Capability Registry：内置 `native` 研究工具、单次结构化 `skill` 和最多三步自审的 `agent`，按稳定 `id@version`、JSON Schema、适用主题、运行表面、上下文权限、模型档位和资源上限注册。每轮先执行研究，再基于同一份冻结上下文并行执行分析与反方挑战；单个能力失败只生成独立失败产物，不中断自然回复。新增商业模式分析、护城河审计和公司论点反驳三个内置能力，并支持从共享本地 `/capabilities/**/*.json` 安全加载声明式扩展；限制文件数量、深度、大小、Schema 子集、输出大小和每轮调用数，拒绝符号链接且不向列表接口暴露 instructions。Skill/Agent 复用现有真实 AI provider 和轻量/标准/深度模型路由，Codex CLI 仍在隔离、禁用工具的进程中运行。所有调用通过可断线重放的强类型事件展示真实阶段、并行状态、provider、model、耗时和错误；结构化成功/失败产物只随 assistant 消息保存一次并在前端独立呈现。声明 `rule_graph` 表面的 `skill`/`agent` 同时注册为规则图精确版本 adapter，激活时校验节点类型、Schema、DAG 和 adapter 可用性，执行轨迹记录节点输入输出。公司研究继续复用 `ResearchPlan` 与 24 小时缓存，并新增独立的社区热点检索工具；所有公司看法、交易和规则写入仍必须通过确认卡交由确定性业务代码执行。
- 重构完整对话运行内核：`ConversationEngine` 入口只保留运行、线程与确认动作 API，主题解析、研究、生成和动作提取迁入独立运行模块；每轮先冻结 `TurnContext`（有效主题、问题、研究计划、模型档位）和 `StepContext`（实际模型上下文与已持久化来源），避免阶段间重新读取可变状态。运行事件改由强类型 `ConversationEvent` 统一生成并保持既有 WebSocket 事件名/JSON 兼容；裸 `JoinHandle` 改为可协作取消的 `TurnTask`，研究、正文生成和动作提取都会响应同一取消信号，超时才强制中止。运行终态使用数据库条件更新，完成、失败、取消和中断中只有第一个状态可以生效，避免取消后又显示完成。主题候选排序和公司后缀剥离使用 Rust 1.96 Clippy 接受的等价写法，保持严格 CI 兼容。
- 放宽长对话的执行与证据预算，避免深度公司分析被过短的保护阈值截断：轻量、标准和深度可见回复硬上限分别调整为 90、240 和 600 秒，轻量/标准与深度确认卡提取分别调整为 120 和 300 秒；正常完成和用户终止仍会立即结束任务。回复模型现在最多接收每类三条研究来源，一手来源每条最多 8,000 字符，独立分析和社区观点每条最多 2,500 字符。存储仍保持每轮最多九条来源、每条最多 8,000 字符及 24 小时研究缓存，防止数据无界增长。
- 将专注护城河分析从摘要清单升级为可审计判断：核心分析目标扩展到约 3,500—4,500 字，每个壁垒机制必须保存事实来源、因果链、竞争者对照、维护成本、反证、信心及升降级/失效条件；没有可观察评分锚点时禁止使用 1—5 分及无依据半分。风险、竞争、反证和“如何攻破”等措辞被视为护城河审计的内部维度，不再误触发综合公司模板。主要击穿路径只保留按概率×影响排序的三至五项，并逐项使用同一字段模板展开攻击者、最低成本步骤、所需资源、公司反制、经济利润传导、领先指标、证伪条件、缺失证据和结论变更点；篇幅不足时减少路径，不能压缩或漏掉最后一条。公司看法投影不再受互相冲突的 500 字符单节限制：护城河、商业模式、财务和其他章节分别受 3,500、3,000、2,000 和 1,200 字符上限约束，单次变更总量最多 9,000 字符，优先压缩重复叙述而非决策证据。深度任务的确认卡提取窗口由 45 秒调整为 90 秒，轻量和标准任务仍为 45 秒。护城河检索同步增加留存、复购、商家经济性、贡献利润、营销效率、质量/退款和竞争反应证据。
- 修复一般线程中自然语言小写证券代码未绑定公司、导致后续详细公司问题不执行研究的问题：`我想投资pdd` 等明确投资/分析/研究语境可安全识别纯字母代码并绑定唯一候选，孤立的 `hi` 等寒暄仍不会被当作证券。
- 修复最近对话只有少量条目时操作菜单被滚动区域裁剪的问题：线程列表现在占满侧栏剩余高度并从顶部排列，归档和删除图标可见且仍支持长列表滚动。
- 显式要求“只根据/仅使用已有本地上下文”的公司问题现在跳过外部检索并使用标准模型，运行状态不再错误显示多年财务抓取；首页刷新后固定打开最近活动线程，不再恢复上一次手工选择。
- 修复真实 Codex CLI 已完成回复却无法生成确认卡的问题：对话动作投影现在使用 CLI 原生 JSON Schema 强制 `summary` / `actions` 外层契约，并在进入业务代码前校验和解码动作载荷；格式异常仍只记录运行告警，不会直接修改公司看法、交易或投资规则。
- 修复普通带标点的对话和“更新公司看法”等指令被误识别为公司简称的问题：只有明确的公司分析结构或唯一高置信公司候选才会进入公司研究；待确认状态也只接收候选序号、证券代码或短公司名，新的完整请求会直接取代旧确认。
- 公司线程存在已确认的公司看法时，右侧上下文栏默认打开“公司看法”，确认新版本后自动展示更新结果；各章节改用安全 Markdown 渲染，开放问题按列表展示。
- 修复研究缓存与运行进度在单连接数据库池中相互等待的问题：研究任务现在可独立推进，运行取消或中断时同步终止，不遗留后台检索。
- 将公司简称解析改为失败关闭：持仓名称、线程公司、证券代码和本地证券目录统一生成候选，只有唯一高置信候选才进入研究。多候选或显式公司请求无法识别时，不再回退到线程原公司，而是使用轻量真实 AI 要求用户确认公司全称或证券代码；该确认轮次不执行外部检索、不读取原线程公司的摘要、看法、来源或规则，也不更新原线程摘要。原请求和候选会随消息持久化，用户回复代码或候选序号后继续执行原请求；仍不唯一时继续确认。前端展示 `subject_clarification` 分级原因。
- 修复公司线程中的跨公司提问污染：例如在绑定 PDD 的线程中输入“分析一下腾讯”，本轮会从持仓名称别名识别 `0700.HK / 腾讯控股`，以腾讯生成研究计划、来源、公司上下文和确认动作，同时保持线程持久主题仍为 PDD。本轮不会再带入 PDD 的线程摘要、历史轮次或公司看法；轮次摘要按有效公司保存和读取，腾讯摘要不会覆盖或污染 PDD 的线程摘要；不同公司的研究查询与缓存身份保持隔离。
- 修复长对话在中等宽度和移动端无法继续输入的问题：`901–1250px` 不再把右侧公司看法放到对话区下方，而是保留线程栏并通过右侧抽屉打开上下文；移动端聊天工作区固定在可用视口内，历史消息仅在消息区滚动，输入框始终保持可见。公司看法、交易和规则确认卡现在跟随提出它们的 assistant 回复显示，不再统一堆在整段对话末尾；已执行/已拒绝卡收为紧凑状态，类型标题跟随界面语言。首次打开或切换线程会定位到最新消息，用户主动上滚查看历史后则不会被后续渲染强行拉回底部。
- 修复详细公司分析中商业模式虽排第一却仍被总字数挤压的问题：综合公司分析改为约 4,000—5,000 字的高密度目标，商业模式至少占实质正文 60%，聚焦商业模式时目标为 3,500—4,500 字且核心分析至少占 75%。商业模式按产品/用户任务、交易全流程与风险归属、变现与利润池、分部单位经济、竞争与攻击者经济、正向复利条件、反向失效与多元思维七层展开，要求分开不同产品/地区经济性并区分事实、推断与未知；详细模式不再被全局“简洁回复”规则覆盖。轻量/标准任务仍为 45/90 秒，深度正文上限由 150 秒调整为 240 秒。真实 PDD 验收正文 8,523 字，其中商业模式 5,678 字、占 66.6%，正文生成 126.7 秒。
- 按巴菲特/芒格公司质量框架补齐经营分析：所有商业模式、护城河和综合公司分析先执行可预测性闸门，将企业评为可预测、部分可预测或不可可靠界定；历史财务基线只代表起点，不再自动触发五年/十年数值外推。只有关键经营变量可界定时才计算经营悲观/基准/乐观情景，否则输出定性情景架构、阻断因素和重新量化所需证据。新增独立的所有者经济性与再投资、管理层/文化/激励/资本配置、财务韧性与毁灭风险审计，覆盖所有者收益及每股口径、维持性/成长性资本开支、稀释、增量投入资本回报、留存收益回报、再投资空间和 great/good/gruesome 分类。“详细/完整/直接分析公司”等通用请求也稳定进入完整框架。SEC Company Facts 新增资本开支、股权激励、稀释加权股数、自由现金流代理值及每股代理值，单位改用结构化类型，并明确代理值不能冒充所有者收益；单年相对相邻年度同时出现 50 倍以上数量级异常时会剔除并标记。年报摘录增加对应经营证据窗口，Tavily 查询按官方/独立/社区焦点拆分，所有来源摘要在入库和缓存前限制为 8,000 字符；public-sources 缓存版本同步升级。深度正文压缩重复方法说明后仍保留全部硬性判断；确认卡提取独立使用标准模型，只有投资系统主题可产生深度规则图。公司回复/投影确定性移除历史估值、交易和投资系统数据，准备与执行公司补丁时也会移除估值变更。CLI 与 OpenAI-compatible 的结构化回复改用 `serde_json` 逐个尝试完整对象，接受代码围栏和尾随说明。
- 将所有公司对话收敛为企业经营分析：模型上下文不再包含持仓汇总、仓位明细、交易历史或投资系统，回复与公司看法确认卡禁止使用股价、行情、市值、估值倍数、目标价、涨跌幅、技术图表、评级、个人持仓盈亏和买卖结论；`valuation_expectations` 暂不由自动公司补丁生成，`thesis` 仅表示公司经营逻辑。产品市场、行业结构、竞争者和市场份额仍属于公司分析。研究 planner 不再生成资本市场查询，Yahoo/TradingView 来源会过滤只谈股票估值、价格和交易技术的内容，public-sources 缓存升级为公司经营口径。
